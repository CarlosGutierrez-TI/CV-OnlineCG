# Comandos Básicos de Git - Ejercicio Práctico

A continuación se define, ejemplifica y simula la salida de los comandos básicos de Git:

1. git --version
* Definición: Muestra la versión instalada de Git en el sistema operativo.
* Ejemplo de uso: git --version
* Salida esperada (simulada):
git version 2.44.0
2. git init
* Definición: Inicializa un repositorio de Git en la carpeta actual, creando el directorio oculto .git.
* Ejemplo de uso: git init
* Salida esperada (simulada):
Initialized empty Git repository in /ruta/a/mi/proyecto/.git/
3. git add
* Definición: Agrega los cambios del directorio de trabajo al área de preparación (staging).
* Ejemplo de uso: git add .
* Salida esperada (simulada):
(sin salida)
4. git commit
* Definición: Registra los cambios del área de preparación en el historial del repositorio local.
* Ejemplo de uso: git commit -m "Primer commit"
* Salida esperada (simulada):
\[master (root-commit) abc1234] Primer commit
3 files changed, 42 insertions(+)
5. git status
* Definición: Muestra el estado actual del árbol de trabajo y del área de preparación.
* Ejemplo de uso: git status
* Salida esperada (simulada):
On branch master
No commits yet
nothing to commit (create/copy files and use "git add" to track)
